Проект містить спеціалізовану wormhole МнК з агрегацією каналів (АК) на System Verilog, що дозволяє визначати довільну кількість фізичних каналів (ФК)
для кожного транку. Іншою особливістю проекту є можливість визначити для кожного генератора трафіку ряд приймачів, куди будуть передаватись потоки даних 
з заданою інтенсивністю, що дозволяє реалізувати довільне розподілення трафіку в мережі. Для введення зазначеного функціоналу використано 
файли “destinations.v” і “links.v”. В файлі “destinations.v” для кожного передавача трафіку, шляхом модифікації массиву destinations, можна визначити ряд 
приймачів, куди будуть передаватися потоки даних і задати інтенсивність введення флітів в МнК для кожного потоку. В файлі “links.v” для кожного транку 
wormhole МнК з АК можна визначити довільну кількість ФК шляхом модифікації масиву links.

Для виконання процедури моделювання необхідно:
1.	Завантажити вихідний код проекту
2.	Розархівувати вміст завантаженого архіву в директорію з назвою “LAG”;
3.	Встановити ModelSim SE версії 6.5b;
4.	Встановити Cygwin версії 1.7;
5.	Відкрити консоль Cygwin і встановити модуль AppConfig версії 1.66, вихідний код якого можна завантажити за посиланням:
http://search.cpan.org/~abw/AppConfig/lib/AppConfig.pm
Встановлення AppConfig відбувається з використанням утиліти make, що входить до пакету Cygwin. 
Makefile генерується програмою “makefile.pl”, що знаходиться серед установочних файлів AppConfig;
6.	Додати в змінну середовища $PATH шлях до каталогу “win32”, що знаходиться всередині папки, куди встановлено ModelSim. 
Наприклад, якщо ModelSim встановлено в папку  "С:\modeltech_6.5”, то в змінну $PATH необхідно додати шлях "С:\modeltech_6.5\win32”;
7.	Відкрити файл router_config/baseline.config і встановити параметри МнК для моделювання;
8.  Налаштувати параметри в файлах “destinations.v” і “links.v” відповідно до параметрів мережі, що моделюється; 
9.	Відкрити консоль Cygwin, перейти в директорію проекту, що розглядається і запустити скриптовий файл “go”
Скриптовий файл “go” виконує запуск моделювання. Після запуску скриптового файлу “go”, 
в консоль Cygwin будуть виведені повідомлення про хід процесу моделювання та результати моделювання.
